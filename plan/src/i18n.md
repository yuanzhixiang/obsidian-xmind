# 多语言文案

## 功能定位

`src/i18n.ts` 维护插件用户界面的本地文案表和语言检测逻辑。当前支持英文 `en` 与简体中文 `zh-CN`，用于 Obsidian 文件视图、源码 viewer、SVG 可访问文案和本地 debug 页。

## 语言检测

- 插件不依赖 Obsidian 官方未公开的 i18n API，也不调用 `app.vault.getConfig()` 探测语言。`Vault.getConfig` 不在公开 `Vault` 类型定义中，属于 Obsidian 内部配置读取通道；在插件 reload、文件视图构造或 Markdown embed 构造期调用，可能因内部 `this.config` 不存在而抛 `Cannot read properties of undefined (reading 'config')`。
- 语言优先读取 `window.localStorage.getItem('language')`，这是社区插件常用的 Obsidian 当前界面语言来源。
- 再回退到 HTML `documentElement.lang` 和浏览器 `navigator.languages[0]`。
- 识别到 `zh`、`zh-CN`、`zh-Hans`、`zh-SG`、`zh-TW`、`zh-HK`、`zh-MO` 时使用简体中文文案；其它语言默认英文。

## 使用方式

- 正式 Obsidian 视图创建 `XMindRenderAdapter` 前调用 `detectXMindLocale()` 并创建 translator。
- `XMindRenderAdapter` 接收可选 `locale`，内部用 `createXMindTranslator()` 渲染工具栏、刷新、大纲、搜索和错误状态文案。
- SVG renderer 接收同一个 `locale`，用于 topic 选择和展开/折叠控件的 `aria-label`。
- layout 接收 `locale`，用于空标题 topic 的 fallback 文案。
- debug 页通过源码 viewer 稳定入口导出的 i18n 函数检测语言，并用 `data-i18n`、`data-i18n-placeholder`、`data-i18n-aria-label` 替换静态 HTML 文案。

## 约束

- 不引入第三方 i18n 依赖，避免增加 Obsidian 发布包体积和扫描面。
- 不保留无入口的编辑、保存或写回文案；当前插件是只读 viewer。
- 新增用户可见文案必须进入 `translations.en` 与 `translations['zh-CN']`，不能只在组件中硬编码。
- 文档、测试脚本名称和内部日志事件名不属于用户界面文案，可以保持现有语言和稳定机器可读名称。
